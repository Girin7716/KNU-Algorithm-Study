# 반도체 설계

## 풀이 방법

예전에 풀어보았던 `전깃줄`문제와 유사하다. 즉, LIS 문제이다.

다만, 여기서는 N <= 40,000 이므로 dp를 활용하여 문제를 풀 경우, O(N^2)으로는 시간초과가 발생한다.

그러므로, 더 효율적으로 LIS를 구하기 위해서는 이분탐색을 사용해야하며, 이를 사용할 경우의 시간 복잡도는 O(NlogN)이 된다.

아래처럼 입력이 주어졌다.
> 4 2 6 3 1 5

그러면 앞에서부터 차례대로 숫자를 검사하며, lis라는 리스트를 만들어준다.

첫 숫자인 4는 lis에 넣어줌
>lis : 4

두번째 숫자인 2와 lis의 마지막(즉,lis[-1]) 숫자인 4를 비교해서 lis[-1]이 더 크니까 숫자 2가 들어가야할 위치를 이분탐색으로 찾는다.
>lis : 2

만약 lis[-1]이 더 작을경우는 lis 리스트에 숫자를 추가해주면 된다.

>lis : 2 6

>lis : 2 3

>lis : 1 3

>lis : 1 3 5

이렇게할경우, 비록 LIS의 원소를 구할 수는 없지만 len(lis)를 통해 길이를 알 수 있다.

찾아보니 원소를 구하는 방법도 있던데 나중에 해보겠다.
